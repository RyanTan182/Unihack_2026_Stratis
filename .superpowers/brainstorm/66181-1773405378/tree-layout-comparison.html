Tree Layout Comparison

Which layout best communicates supply chain hierarchy?

A — Top-Down Hierarchy

Root at top, tiers flow downward. Natural reading order. Clear tier separation. Best for showing depth.

B — Left-to-Right Flow

Root on left, branches flow right. Better horizontal space usage in a wide panel. Reads like a process flow.

My take: For a supply chain, left-to-right (B) actually works better than top-down. The expanded panel is wider than it is tall, and L→R naturally reads as "product → subsystem → component → material" — a process flow. Top-down requires more vertical scrolling for deep trees. But both work — it's a preference call.
